# 배운점
- 백트래킹 오랜만에 풀어봐서 감을 다시 잡은 느낌. bfs로는 풀 수 없는 문제임(사이클 판단 불가)
- 시간초과가 뜰 줄 몰랐는데... 방심했음. dp 느낌쓰로 해결(사이클 판단이랑 상태값 통일 해도 될 듯?)
- dp 배열 크기 판단 실수함

\+
- DP 느낌쓰가 아니고 DP로 풀 수가 있음
- 다만 관점의 전환이 필요함. 일단 잘 생각해보면 해당 위치까지 가는 경로는 중요하지 않다는 걸 알 수 있음
- 이게 바로 DP의 신호. like 뮤탈리스크 문제
- 경로가 아니라 값에 집중하기 시작하면 DP적 풀이가 가능하다.
- 그렇기 때문에 int dfs(y, x)를 해당 위치에서 이동할 수 있는 최대 횟수로 정의하면 dp 풀이가 됨.
